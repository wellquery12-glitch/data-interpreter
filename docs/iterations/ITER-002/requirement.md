# ITER-002 需求文档（待确认）

## 1. 迭代名称
多主题商业分析能力扩展（运营/财务/客户）

## 2. 背景与目标
`ITER-001` 已完成销售主题自动分析能力，并在 `2026-03-11` 完成验收相关变更。
本次迭代从 `2026-03-11` 重新启动，聚焦补齐其余主题能力，提升商业分析覆盖面。

## 3. 本轮目标（草案）
1. 完成运营主题自动分析模板与建议结构化输出。
2. 完成财务主题自动分析模板与建议结构化输出。
3. 完成客户主题自动分析模板与建议结构化输出。
4. 保持导出能力（Markdown/PDF）与页面流程一致，不破坏已上线行为。

## 4. 用户待确认范围
1. 三个主题中，是否需要优先级顺序（例如先运营，再财务，再客户）。
2. 每个主题是否都要求图表输出（若要求，明确最少图表类型）。
3. 每个主题是否都要求 `impact_estimation` 必须量化（还是允许定性）。
4. 本轮是否包含历史数据同比/环比分析。

## 5. 功能需求（草案）
### FR-1 主题模板扩展
- 当主题为运营/财务/客户时，系统执行对应模板分析逻辑。
- 返回结构与销售主题保持一致：`finding`、`evidence`、`suggestion`、`priority`、`impact_estimation`。

### FR-2 主题结果可读性
- 每个主题结果至少包含 3 条建议（字段不足时给出可分析范围说明）。
- 输出中需体现当前主题关注点（如运营效率、财务健康、客户价值/流失）。

### FR-3 导出与页面一致性
- `/insights/auto` 与 `/insights/export` 在新主题下行为一致。
- 页面端对三类主题不新增额外操作门槛。

## 6. 非功能要求
1. 不影响现有 `/ask` 功能。
2. 不影响 `ITER-001` 已有销售主题能力。
3. 缺少关键字段时，返回可读提示而非中断报错。

## 7. 验收标准（草案）
1. 运营/财务/客户三个主题均可正常自动分析。
2. 三个主题均可导出 Markdown 与 PDF。
3. 每个主题至少具备 1 组新增测试，且关键回归通过。
4. 用户按验证清单检查通过后，进入合并阶段。

## 8. 风险与依赖
1. 不同数据集字段命名差异较大，可能导致主题识别偏差。
2. 若样例数据不足，`impact_estimation` 量化可能退化为定性。
3. 若新增图表策略，前端展示可能需要同步调优。
