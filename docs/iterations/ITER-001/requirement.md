# ITER-001 需求文档（已确认）

## 1. 迭代名称
商业智能分析与建议引擎（面向商业用户）

## 2. 背景与目标
当前系统已经支持“上传数据 -> 提问 -> 生成分析结果”。
本次迭代新增“自动商业分析 + 商业建议生成”能力，使商业用户无需手工提问也能得到可执行建议。

## 3. 用户已确认范围
1. 优先主题：**销售分析**。
2. 其他主题（运营/财务/客户）暂列入项目待办，不在本次实现。
3. 导出格式：支持 **Markdown** 与 **PDF**。
4. 影响预估：根据数据情况，支持“量化 + 定性”混合输出。
5. 展示方式：使用**新页面**展示自动分析报告。

## 4. 功能需求
### FR-1 自动分析触发
- 用户选中数据集后可点击“自动分析（销售）”。
- 无需输入问题，系统自动完成分析。

### FR-2 销售主题自动分析
- 自动识别并使用销售相关字段（金额、时间、客户/交易对象、产品等）进行分析。
- 输出关键发现（至少 3 条，字段不足时给出可分析范围说明）。

### FR-3 建议生成
- 每条建议必须包含：`finding`、`evidence`、`suggestion`、`priority`、`impact_estimation`。
- `impact_estimation` 按数据可用性自动给出量化或定性结论。

### FR-4 报告导出
- 支持导出 Markdown 文件。
- 支持导出 PDF 文件。

### FR-5 页面展示
- 新增独立页面用于展示自动分析报告、导出按钮和结构化建议列表。

## 5. 非功能要求
1. 保持现有 `/ask` 功能行为不变。
2. 失败时返回可读错误信息，不得静默失败。
3. 单次分析应在可接受时间内完成（中等规模数据默认目标 60 秒以内）。

## 6. 验收标准
1. 新页面可选数据集并触发自动分析。
2. 自动分析输出结构完整，建议包含证据与优先级。
3. 可成功导出 Markdown 与 PDF。
4. 缺少关键字段时系统给出可分析范围说明。
5. `/ask` 原有能力回归通过。

## 7. 待办（不在本次迭代实现）

- 财务主题自动分析模板
- 客户主题自动分析模板
